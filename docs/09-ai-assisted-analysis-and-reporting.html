AI-Assisted Analysis and Reporting

Published

Jun 2026

  • ID: AI-09
  • Type: Systems Guide
  • Audience: Analysts, researchers, educators, students, technical teams, and workflow designers
  • Theme: AI can support analysis and reporting, but human judgment must remain responsible for interpretation, evidence, and claims

AI can help people move from raw information to clearer analysis, better summaries, cleaner documentation, and more polished reports.

But AI-assisted analysis is not the same as AI-generated truth.

An AI tool can help organize evidence. It can suggest patterns. It can draft explanations. It can create first-pass summaries. It can help translate technical outputs into plain language.

The human remains responsible for checking the evidence, interpreting the results, deciding what claims are justified, and communicating uncertainty honestly.

This chapter explains how to use AI inside analytical and reporting workflows without weakening reproducibility, traceability, or scientific judgment.

The CDI position is simple:

AI can assist analysis.

AI can assist reporting.

But conclusions must remain evidence-led and human-reviewed.

Learning Outcomes

By the end of this chapter, you should be able to:

  • identify appropriate uses of AI in analysis and reporting;
  • distinguish evidence, interpretation, and communication;
  • design an AI-assisted reporting workflow that remains human-led;
  • use structured inputs to improve AI-generated summaries;
  • review AI-assisted reports for unsupported claims;
  • connect analysis and reporting back to the Human-AI-Human workflow.

Why AI-Assisted Reporting Matters

Many analytical projects fail after the analysis is complete.

The results exist, but they are not communicated clearly.

The tables are correct, but the interpretation is weak.

The plots are useful, but the claims are too broad.

The report is written, but it does not explain what decisions the results support.

AI can help with this translation layer.

It can help turn structured results into readable summaries. It can help create outlines. It can compare draft interpretations against source tables. It can help explain methods for different audiences.

However, this only works well when the AI is grounded in the actual project outputs.

AI should not invent an analysis.

It should help explain an analysis that already exists.

The Human-AI-Human Reporting Loop

AI-assisted reporting follows the same CDI pattern used throughout this guide.

Human prepares the evidence, question, audience, and reporting goal
        ↓
AI assists with structure, summary, explanation, or draft language
        ↓
Human verifies evidence, revises claims, documents uncertainty, and approves the final report

The human starts with the analytical context.

The AI helps create a draft or supporting explanation.

The human reviews the output against the data and decides what can be said.

This matters because reporting is not only writing.

Reporting is interpretation.

Interpretation requires accountability.

Analysis, Interpretation, and Communication

AI-assisted reporting becomes safer when these three layers are separated.

Layer Main question Human responsibility AI support
Analysis What was computed? Verify data, methods, and outputs Summarize tables or workflow steps
Interpretation What does it mean? Decide what claims are justified Suggest possible explanations or caveats
Communication How should it be explained? Match message to audience and context Draft summaries, outlines, and plain-language text

Problems happen when AI skips directly from data to conclusion.

A responsible workflow keeps the layers visible.

Appropriate Uses of AI in Analysis Workflows

AI can support analysis without replacing analytical judgment.

Useful tasks include:

  • summarizing a results table;
  • explaining column meanings;
  • drafting an analysis plan;
  • identifying missing documentation;
  • suggesting quality-control checks;
  • helping interpret plots;
  • turning technical outputs into narrative explanations;
  • drafting report sections from verified results;
  • creating reviewer checklists;
  • rewriting technical text for a different audience.

These uses are strongest when the AI is given concrete inputs.

For example, a useful input might include:

  • the research or business question;
  • the analysis goal;
  • the dataset description;
  • the results table;
  • the plot or figure description;
  • the methods summary;
  • the intended audience;
  • the claims that are allowed;
  • the claims that are not allowed.

The more grounded the input, the more reviewable the output.

Risky Uses of AI in Analysis Workflows

Some uses require extra caution.

AI should not be used carelessly to:

  • invent results that were not computed;
  • generate citations without verification;
  • make causal claims from descriptive data;
  • hide uncertainty;
  • replace statistical review;
  • decide clinical, legal, financial, or policy action without expert oversight;
  • summarize sensitive data without privacy safeguards;
  • produce final reports that no human has checked.

AI can make unsupported claims sound polished.

That is why polish is not evidence.

A well-written AI paragraph can still be wrong.

A CDI Pattern for AI-Assisted Reports

A useful AI-assisted report should move from evidence to interpretation to action.

Question
  What is the report trying to answer?

Inputs
  What data, tables, plots, notes, or documents were used?

Methods
  How were the outputs produced?

Results
  What was observed?

Interpretation
  What do the results suggest?

Limitations
  What should not be overclaimed?

Recommended next steps
  What should the human team do next?

This structure helps prevent the report from becoming a loose narrative.

It also helps reviewers check whether every claim is connected to evidence.

Example: Turning a Results Table into a Report Summary

Suppose an AI evaluation workflow creates this file:

data/results/ai-evaluation-summary.csv

A human should inspect the file first.

cat data/results/ai-evaluation-summary.csv

Then the human can ask AI for a summary using a grounded prompt.

You are helping write a short analytical summary for a CDI AI Systems guide.

Use only the results below.
Do not invent additional results.
Do not claim that the workflow is production-ready.
Separate observations, interpretation, and limitations.

Project context:
This table summarizes example AI task evaluations across criteria such as clarity, grounding, completeness, risk level, and review status.

Audience:
Learners building responsible AI workflows.

Results table:
[paste table here]

Write:
1. a short results summary,
2. a brief interpretation,
3. two limitations,
4. two recommended next steps.

This prompt is safer than asking:

Write a report about my AI system.

The better prompt gives the AI context, boundaries, evidence, and reporting structure.

Reporting from Plots and Figures

AI can help describe a figure, but the human should verify the description.

A good figure-summary prompt includes:

Figure title:
AI Evaluation Summary

Figure source:
data/reports/ai-evaluation-summary.png

Data source:
data/results/ai-evaluation-summary.csv

What the figure shows:
[paste your own description]

Task:
Write a concise figure caption and interpretation.
Separate what is directly visible from what is inferred.
Do not make claims beyond the figure.

This avoids treating visual interpretation as automatic truth.

The human still checks the figure, the source data, and the written claim.

AI-Assisted Reporting Checklist

Before using AI to assist with reporting, ask:

  • What evidence is the report based on?
  • Has the evidence been checked?
  • What is the intended audience?
  • What decisions might the report influence?
  • What claims are supported?
  • What claims are not supported?
  • Are uncertainty and limitations included?
  • Are sensitive details protected?
  • Can another person trace the report back to the inputs?
  • Has a human reviewed the final version?

A report that cannot be traced back to its evidence should not be treated as reliable.

Common Failure Modes

AI-assisted analysis and reporting can fail in predictable ways.

Failure mode What happens Prevention
Unsupported claim AI states something not shown in the data Require evidence-linked summaries
Overgeneralization A small example becomes a broad conclusion Add scope and limitation statements
Method confusion AI mixes up tools, models, or data steps Provide a methods summary
Citation weakness AI invents or misuses references Verify all references manually
Missing uncertainty AI writes with too much confidence Ask for limitations and caveats
Audience mismatch Output is too technical or too vague Define the intended audience
Privacy leakage Sensitive information appears in outputs Remove or mask sensitive inputs

These are not reasons to avoid AI.

They are reasons to design the workflow properly.

The Role of Scripts in Reporting

In a CDI guide, AI-assisted reporting should connect to files generated by scripts.

For example, the scaffold includes scripts that create and summarize an example AI task log.

bash scripts/bash/01-create-example-ai-task-log.sh

python scripts/python/02-evaluate-ai-output-quality.py \
  data/example/ai-task-log.csv \
  data/results

python scripts/python/03-build-prompt-registry.py \
  data/example/ai-task-log.csv \
  data/results

Rscript scripts/R/04-plot-evaluation-summary.R \
  data/results/ai-evaluation-summary.csv \
  data/reports

These commands produce structured outputs that can support reporting.

data/example/ai-task-log.csv
data/results/ai-evaluation-summary.csv
data/results/prompt-registry.csv
data/reports/ai-evaluation-summary.png

The key point is that AI should not be asked to report from memory.

It should report from traceable project outputs.

A Reproducible AI Reporting Workflow

A reproducible reporting workflow should make the path from input to report visible.

Project question
        ↓
Input data, notes, prompts, or task logs
        ↓
Scripted summaries and checks
        ↓
AI-assisted draft explanation
        ↓
Human review against evidence
        ↓
Final report with limitations and next steps

This pattern connects AI Systems to the broader CDI ecosystem.

The goal is not simply to write faster.

The goal is to write more clearly while preserving evidence, judgment, and accountability.

Key Takeaways

AI-assisted analysis and reporting are most useful when the evidence is already organized.

AI can help summarize, structure, explain, and revise.

It should not replace data validation, statistical reasoning, domain expertise, or final human review.

A responsible report separates results, interpretation, limitations, and next steps.

The Human-AI-Human workflow keeps the reporting process grounded: the human prepares the evidence, AI assists with drafting or explanation, and the human verifies and approves the final communication.
